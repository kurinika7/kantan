# 簡単帳簿（PWA）

スマホ・PCのブラウザで動く家計簿/簡易経理アプリ。レシートや納品書を撮影すれば自動で読み取り（OCR）、手入力にも対応。データはすべて端末内（IndexedDB）に保存され、外部送信されません。

## 主な機能

- **レシート/納品書の自動読み取り** — 撮影またはファイルアップロードした画像から、店名・日付・合計金額を自動抽出（Tesseract.js 日本語OCR）
- **手入力モード** — 種別（支出/収入）・日付・金額・店名・カテゴリ・メモを直接入力
- **店舗カテゴリ分け** — コンビニ／スーパー／飲食店／ドラッグストア／公共料金／オンライン通販 など
- **商品カテゴリ分け** — 食品／日用品／衣類／家電／交通費／光熱費 など
- **商品の仕分け** — 1枚のレシートに含まれる複数商品を個別に記録（各商品にカテゴリ付与可）
- **グラフ化** — 商品カテゴリ別ドーナツチャート、店舗カテゴリ別棒グラフ、店舗別ランキング、月別収支
- **CSV/JSON エクスポート・インポート** — バックアップや乗り換えが容易
- **完全オフライン動作** — Service Worker でキャッシュ済み

## セットアップ

### A. すぐ試す（PCで開く）

1. このフォルダの `index.html` をダブルクリック → ブラウザで開きます
2. すべての機能が使えます（OCR・カメラを除く一部はローカルファイルでも動作）

> ⚠️ **OCR を使う場合**は、ブラウザのセキュリティ制約のため`file://` ではなく `http://` 経由で開く必要があります（下記の B か C の方法）

### B. ローカルサーバーで起動（推奨／開発・テスト用）

ターミナルで以下を実行：

```bash
cd 簡単帳簿
python3 -m http.server 8000
# または
npx serve .
```

スマホから同じWi-Fi内でアクセスする場合：

```bash
# PCのIPアドレスを確認 (例: 192.168.0.10)
# スマホのブラウザで http://192.168.0.10:8000 を開く
```

### C. 公開ホスティングにデプロイしてホーム画面に追加（実用）

1. このフォルダ全体を以下のいずれかにアップロード：
   - **Netlify Drop** — https://app.netlify.com/drop （フォルダをドラッグ＆ドロップでURL発行）
   - **Vercel** — https://vercel.com (CLIまたはGitHub連携)
   - **GitHub Pages** — リポジトリの `Settings → Pages`
   - **Cloudflare Pages** — https://pages.cloudflare.com
2. 発行されたURLをスマホで開く
3. **iPhone (Safari)**: 共有ボタン → 「ホーム画面に追加」
4. **Android (Chrome)**: ︙メニュー → 「アプリをインストール / ホーム画面に追加」

これでスマホのホーム画面にアイコンが追加され、ネイティブアプリのように起動できます（オフライン動作対応）。

## 使い方

### レシートから入力する

1. 下部メニューの「**追加**」タブを開く
2. 「📷 撮影」または「📁 ファイル選択」をタップ
3. レシートが解析され、店名・日付・金額が自動入力されます
4. 必要に応じて修正してから「**保存**」

### 手入力する

1. 「追加」タブで、画像なしでも直接フォームに入力できます
2. 種別（支出/収入）→ 日付 → 金額 → 店名 → カテゴリ → メモ
3. 「保存」をタップ

### 商品を個別に仕分けする

「商品の仕分け（任意）」セクションで「+ 行を追加」をタップし、商品名・金額・カテゴリを行ごとに入力。1枚のレシートに食品と日用品が混在する場合などに便利です。

### グラフを見る

「グラフ」タブで期間（今月/先月/直近3ヶ月など）を切り替えながら、商品カテゴリ別・店舗カテゴリ別・店舗ランキング・月別収支を確認できます。

### バックアップ

「設定」タブの「💾 JSON書き出し」で全データをファイルに保存できます。機種変更時などは、新端末で同じアプリを開いて「📥 JSON取り込み」で復元してください。

## 技術スタック

- HTML / CSS / Vanilla JavaScript（フレームワーク非依存）
- IndexedDB（端末内ストレージ）
- [Tesseract.js 5.x](https://github.com/naptha/tesseract.js)（ブラウザ内OCR、日本語）
- [Chart.js 4.x](https://www.chartjs.org/)
- Service Worker（オフライン対応）
- Web App Manifest（ホーム画面追加・スタンドアロン表示）

## プライバシーについて

- 撮影/アップロードした画像、OCR結果、明細データはすべて**お使いの端末内（IndexedDB）**にのみ保存されます
- 外部サーバーへの送信は一切ありません（OCRもブラウザ内で完結）
- 初回読み込み時のみ、CDN（jsdelivr.net）から Chart.js / Tesseract.js を取得します

## 既知の制限

- Tesseract.js の日本語OCRは精度が中程度です。複雑なレイアウトや手書き、薄いレシートでは誤認識が起きることがあります → 手で修正してください
- iOS の Safari では、アプリをホーム画面に追加してから「機内モード→オンに戻す」をしないと PWA として安定動作しないことがあります
- 大量の画像を保存すると IndexedDB の容量上限に達する場合があります（端末・ブラウザにより数百MB〜数GB）
